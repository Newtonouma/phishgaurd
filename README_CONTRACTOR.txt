This document covers everything a contractor needs to build the Windows .exe
and package the Chrome extension. All source files are provided in the zip.

--------------------------------------------------------------------------------
  PROJECT OVERVIEW
--------------------------------------------------------------------------------

PhishGuard is a two-part phishing detection system:

  PART A — Windows Desktop Application (.exe)
    A Tkinter-based desktop dashboard that classifies emails as phishing or
    legitimate using 5 trained ML models. Includes Outlook inbox integration,
    XAI charts (SHAP, LIME, Decision Tree path), and a chatbot interface.

  PART B — Chrome Browser Extension
    Injects a real-time analysis panel into Gmail. Sends email content to a
    local Python API server (api_server.py) and displays the verdict inline.

Both parts are already fully coded. The contractor's job is to:
  1. Build the .exe using PyInstaller (instructions below)
  2. Package and verify the Chrome extension loads correctly
  3. Test both deliverables end-to-end
  4. Return the finished files

--------------------------------------------------------------------------------
  DELIVERABLE 1 — WINDOWS .EXE BUILD
--------------------------------------------------------------------------------

WHAT IT SHOULD DO:
  Double-clicking PhishGuard_UWS.exe opens a full desktop application with:
  - A dark navy sidebar with 7 navigation pages
  - Dashboard with KPI cards and live charts
  - Analyse Email page: paste any email, get instant ML verdict + explanation
  - Outlook Inbox integration: import and classify live Outlook emails
  - Model Metrics page: accuracy, F1, AUC, confusion matrices per model
  - XAI Charts: ROC curves, SHAP feature importance, k-fold CV plots
  - Dataset Management: load CSV files to re-train models
  - Settings and About page

REQUIREMENTS:
  - Windows 10 or 11, 64-bit
  - Python 3.11 or 3.12 (64-bit) — MUST be 64-bit
  - All dependencies installed (see below)
  - PyInstaller 6.6.0

SETUP STEPS:
  1. Install Python 3.11 64-bit from python.org
     During install: tick "Add Python to PATH"

  2. Create a virtual environment:
       python -m venv phish_env
       phish_env\Scripts\activate

  3. Install all dependencies:
       pip install scikit-learn==1.4.2
       pip install pandas==2.1.4
       pip install numpy==1.26.4
       pip install matplotlib==3.8.4
       pip install seaborn shap lime flask flask-cors
       pip install openpyxl pywin32 joblib
       pip install pyinstaller==6.6.0

  4. Navigate to the PhishGuard folder:
       cd C:\path\to\PhishGuard

  5. Run the build script:
       python build_exe_phishguard.py

  6. Wait 5–10 minutes. The finished exe will be at:
       PhishGuard\dist\PhishGuard_UWS.exe

EXPECTED BUILD OUTPUT:
  - File: dist\PhishGuard_UWS.exe
  - Size: approximately 250–400 MB (normal for PyInstaller + ML libraries)
  - No installation required on target machine — fully self-contained

VERIFICATION CHECKLIST (test before returning):
  [ ] Double-clicking the exe opens the app without errors
  [ ] The sidebar loads with all 7 menu items
  [ ] The app starts and shows "Demo data trained. Ready." in the status bar
      within 20 seconds (this is the fallback demo training — expected)
  [ ] Pasting a phishing-style email in Analyse Email and clicking Analyse
      returns a PHISHING verdict with confidence score
  [ ] Pasting a normal business email returns a LEGITIMATE verdict
  [ ] All 5 model chips appear in the results row (LR, DT, SVM, NB, RF)
  [ ] XAI Charts page renders without crashing
  [ ] Model Metrics page shows "Train All Models" button
  [ ] No console errors or crash dialogs on any page

COMMON BUILD ISSUES AND FIXES:
  - "32-bit Python" error: uninstall Python, reinstall the 64-bit version
  - "Missing app.py": the file is named app_phishguard.py in source —
    rename it to app.py before running the build script
  - "ModuleNotFoundError: phishing_pipeline": ensure phishing_pipeline.py
    is in the same folder as app.py when building
  - SHAP or LIME import errors during build: add
    --collect-all=shap --collect-all=lime to the build command in
    build_exe_phishguard.py (already included for sklearn and matplotlib)
  - Antivirus blocking the exe: this is normal for PyInstaller builds.
    The exe is safe — add an exclusion or submit for signing if needed.

--------------------------------------------------------------------------------
  DELIVERABLE 2 — CHROME EXTENSION
--------------------------------------------------------------------------------

WHAT IT DOES:
  When loaded in Chrome and a user opens Gmail (mail.google.com), the extension
  automatically reads the open email and sends it to the local API server
  running on localhost:5000. The API classifies it and returns a verdict. The
  extension displays a PhishGuard panel above the email showing:
  - PHISHING or LEGITIMATE verdict with confidence percentage
  - Per-model breakdown (all 5 models)
  - AI explanation of why it was flagged
  - Links to the full desktop dashboard

  If the API server is offline, the extension falls back to a built-in
  rule-based checker (urgency keywords, URL count, caps ratio).

FORMAT:
  The extension is a FOLDER, not an installer. Chrome loads it directly.
  It does NOT need to be uploaded to the Chrome Web Store for local use.
  It uses Manifest Version 3 (MV3) — the current Chrome standard.

HOW TO LOAD IT IN CHROME:
  1. Open Chrome
  2. Go to: chrome://extensions
  3. Toggle ON "Developer mode" (top-right)
  4. Click "Load unpacked"
  5. Select the extension/ folder from the zip
  6. The PhishGuard card appears — extension is now active

HOW TO PACKAGE IT FOR HANDOFF:
  The extension/ folder IS the deliverable. Zip it as extension.zip or
  hand it back as a folder. Do not flatten the folder structure — all files
  must stay in their relative positions.

  If submitting to Chrome Web Store in future:
  - Zip the contents of extension/ (not the folder itself)
  - Upload to Chrome Developer Dashboard at chrome.google.com/webstore/devconsole
  - This requires a one-time $5 developer registration fee

VERIFICATION CHECKLIST (test before returning):
  [ ] Extension loads in Chrome without errors (no red error icon)
  [ ] PhishGuard shield icon appears in Chrome toolbar
  [ ] Clicking the icon shows the popup with API Status and Analyse button
  [ ] Opening Gmail shows no console errors in DevTools
  [ ] With api_server.py running: opening an email triggers the analysis panel
  [ ] The panel shows verdict, confidence, and per-model chips
  [ ] Dismiss button closes the panel
  [ ] With api_server.py stopped: extension shows "Offline" badge in popup
      and still analyses using the rule-based fallback

HOW THE EXTENSION COMMUNICATES WITH THE DESKTOP APP:
  The extension does NOT talk to the .exe directly. It talks to api_server.py
  which must be running separately as a Python process on port 5000. The .exe
  is the visual dashboard — the API server is the ML backend. Both use the
  same trained model file.

  Architecture:
    Gmail (Chrome) → extension → api_server.py → phishing_pipeline.py → verdict
    Outlook (desktop) → PhishGuard_UWS.exe → phishing_pipeline.py → verdict

--------------------------------------------------------------------------------
  WHAT THE CLIENT WILL PROVIDE AFTER TRAINING
--------------------------------------------------------------------------------

Once the client runs their Google Colab notebook on real email datasets,
they will provide three files to drop into the PhishGuard/ root folder:

  phishguard_model.joblib    (approx 50–200 MB — all 5 trained models)
  model_results.json         (small — performance metrics)
  phishguard_dataset.csv     (large — merged training dataset)

These should be placed alongside app.py and api_server.py:
  PhishGuard\phishguard_model.joblib
  PhishGuard\model_results.json
  PhishGuard\phishguard_dataset.csv

When these files are present:
  - The desktop app loads the real models on startup instead of demo training
  - The API server loads all 5 models including Decision Tree path explainer
  - Accuracy improves significantly over the demo fallback

The exe build can be completed WITHOUT these files. The build script does
not bundle the model — it is loaded at runtime from the same folder as the exe.
This means the client can update the model file without rebuilding the exe.
